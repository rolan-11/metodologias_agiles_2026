# Ejercicio Nro: 2

## Enunciado
¿Qué es para vosotros el desarrollo de software?
Vamos a contestar entre todos algunas preguntas, para conocer vuestras opiniones y experiencias:
* ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
* ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
* ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
* ¿Cuándo se ha desarrollado un buen software?

## Resolución

* **Ejemplos de desarrollo en los que participé y que conozco:** Participé en varios trabajos de la facu. Por ejemplo, hace poco armamos un dashboard de ventas web usando .NET y SQL Server para mostrar rankings y métricas. También estuve armando consultas SQL bastante pesadas para un sistema de información de vuelos. Ejemplos que conozco... básicamente cualquier app que usamos en el celu todos los días, o los sistemas de stock y facturación que se usan en los locales comerciales.
* **El papel del desarrollador:** Para mí, el desarrollador es el que agarra lo que el cliente necesita y lo baja a la realidad armando el código. Básicamente es pensar la lógica, estructurar bien las bases de datos para que nada explote a futuro, y programar las funciones para que el sistema ande como debe.
* **Otros papeles importantes:** Sí, obvio. En un proyecto real no es solo tirar código. Tenés a los analistas funcionales que relevan los requisitos, los de QA que se encargan de probar todo y buscar bugs antes de entregarlo, los que organizan el proyecto (Project Managers) y los de redes o infraestructura que configuran los servidores y los protocolos para que el sistema esté online.
* **¿Cuándo se ha desarrollado un buen software?:** Un software es bueno cuando realmente le soluciona la vida al usuario y es fácil de usar. No sirve de nada armar un sistema rapidísimo o re complejo si el usuario final no lo entiende. Y por el lado técnico, es bueno cuando el código está prolijo y se puede escalar; porque si el día de mañana hay que modificar algo y el código es un desastre, es un dolor de cabeza
